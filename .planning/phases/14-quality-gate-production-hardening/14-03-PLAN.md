<objective>
Make the GNOME Shell extension theme-aware by leveraging GNOME 46's built-in dual-stylesheet mechanism: dark and light stylesheets with WCAG AA-compliant provider colors.

Purpose: The extension currently forces a dark theme regardless of system preference. On Adwaita Light, it looks like a foreign dark widget in a light desktop. This fix makes it native to both themes.
Output: Three CSS files (dark, light, fallback) and simplified extension.js without manual stylesheet loading.
</objective>

<!-- GNOME Shell 46 dual-stylesheet mechanism (verified from research) -->

Resolution priority in extensionSystem.js:
1. `${global.sessionMode}-${variant}.css` (e.g., "user-dark.css")
2. `stylesheet-${variant}.css` (e.g., "stylesheet-dark.css") -- THIS IS WHAT WE USE
3. `${global.sessionMode}.css` (e.g., "user.css")
4. `stylesheet.css` (universal fallback)

Variant resolution via Main.getStyleVariant():
- `org.gnome.desktop.interface color-scheme` = 'prefer-dark' → variant = 'dark'
- `org.gnome.desktop.interface color-scheme` = 'prefer-light' → variant = 'light'

Auto-reload: St.Settings 'notify::color-scheme' → _reloadExtensionStylesheets()
CRITICAL: Reload ONLY works for files ending in -dark.css or -light.css

CONFLICT: Current extension.js manually loads stylesheet.css in _loadStylesheet().
GNOME Shell already loads the stylesheet automatically before enable() is called.
Manual loading MUST be removed to avoid double-loading conflicts.

<!-- Current extension.js manual loading (lines 93-116 — REMOVE) -->
```javascript
_loadStylesheet() {
  if (this._stylesheetFile) return;
  const stylesheetFile = this.dir.get_child('stylesheet.css');
  if (!stylesheetFile.query_exists(null)) return;
  const theme = St.ThemeContext.get_for_stage(global.stage).get_theme();
  theme.load_stylesheet(stylesheetFile);
  this._stylesheetFile = stylesheetFile;
}

_unloadStylesheet() {
  if (!this._stylesheetFile) return;
  const theme = St.ThemeContext.get_for_stage(global.stage).get_theme();
  theme.unload_stylesheet(this._stylesheetFile);
  this._stylesheetFile = null;
}
```
