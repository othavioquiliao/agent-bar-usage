## Design Context

### Users
Agent Bar Ubuntu is for a broader Ubuntu and developer audience that wants fast, reliable visibility into AI agent usage directly from the Ubuntu top bar. The primary job is passive monitoring first: users should be able to glance at quota and health state without opening a full app, then click for a compact second layer with only the most relevant supporting details.

The product should help users notice meaningful changes quickly, especially quota consumption, degraded provider state, and actionable next steps when something breaks. The interface should support fast scanning, not reading-heavy inspection.

### Brand Personality
Voice and tone should feel like: Get Shit Done.

Three-word personality:
- Precise
- Calm
- Capable

Emotionally, the interface should create confidence and clarity, with strategic color used to draw attention only when it helps decisions. It should feel native to Ubuntu and developer workflows rather than loud, corporate, or over-designed.

### Aesthetic Direction
The preferred direction is minimal but playful. The current GNOME-native utility feel is valid, but the visual system should borrow selected cues from `qbar` where appropriate, especially provider branding assets such as the existing vendor icons in `qbar/icons/`.

The UI should avoid dense blocks of explanatory text in menus. When the user expands the menu, it should prioritize specific data with compact visual structure, especially progress bars and short labels, instead of verbose status prose. Color should be used sparingly and intentionally, with restrained accents rather than highly saturated or decorative palettes.

Reference cues:
- `qbar` provider icons and compact status presentation
- Native Ubuntu/GNOME affordances for layout and interaction

Anti-direction:
- Text-heavy popovers or menus
- Overly colorful dashboards
- Generic SaaS-like admin-panel styling

Theme guidance:
- No strong explicit light/dark requirement was given
- Prefer respecting host-platform theming where practical
- Keep accents restrained in either theme

### Design Principles
- Optimize for glanceability first. The top-bar state must be understandable in seconds without opening deeper UI.
- Prefer signal over explanation. Expanded menus should show compact, high-value metrics and progress bars instead of verbose paragraphs.
- Use color strategically, not decoratively. Neutral surfaces should dominate, with color reserved for provider identity, severity, and progress emphasis.
- Preserve native credibility. Interactions and structure should feel at home in Ubuntu/GNOME, even when borrowing selected visual ideas from `qbar`.
- Keep playfulness controlled. Small moments of personality are welcome, but the interface should stay precise, calm, and operational.
- Design for accessibility by default. Information should not rely on color alone, motion should stay restrained, and contrast/readability should support everyday developer use.
